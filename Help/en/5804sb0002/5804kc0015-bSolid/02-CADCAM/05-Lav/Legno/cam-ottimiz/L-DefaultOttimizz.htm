Type of optimisation

TypeOfOptimisation

Used to select the type of optimisation.

The three options are linked to the fields in the Typical optimisation data and TS algorithm sections.

By work list

Used to arrange the machining operations in order, following the sequence indicated in the source program.

Note only for machines with two independent operating sections (e.g. Excel):

To take full advantage of the parallel machining of the two operating sections, the system always tries to optimise on the basis of the time. If you want to use this type of option, you must therefore insert ISOAcronym of the "International Standardisation Organisation", that determines technological and industrial standards. File format containing the programming language of the numerical control machine. rows (even empty ones) between the machining operations of the program to be excluded from the default optimisation.

By Time

Used to minimise machining times.

By type of machining operation

Used to prioritise the machining operations on the basis of the type. Indicate the sequence in the Types of machining operations section.

By work list and head boring

Used to prioritise the boring operations that can be optimised, grouping them together at the start of the program or with the non-optimisable operations so as to make the least possible number of descents.

Note only for machines with two independent operating sections (e.g. Excel):

To take full advantage of the parallel machining of the two operating sections, the system always tries to optimise on the basis of the time. If you want to use this type of option, you must therefore insert ISO rows (even empty ones) between the machining operations of the program to be excluded from the default optimisation.

By tool

Used to prioritise the machining operations, following the tool sequence defined in the Tools section.

Accuracy of test to check spindle step and bit diameter

Precis

Tolerance to select tools on the basis of their diameter or the distance between the spindle centres. Recommended values: 0 or 1.

Typical optimisation data
Direction

TypeOfXDir

Type of arrangement based on the direction. Choose the option in the drop-down menuList of options or instruments which expand vertically or horizontally when you click on the drop-down buttons..

None

Data fieldA defined area containing data or commands (data field or command field). disabled.

X+

Forces the optimiser to arrange boring operations so that the processing centre always moves in the positive X direction.

X-

Forces the optimiser to arrange boring operations so that the processing centre always moves in the negative X direction.

Change tool order

ChangeToolOrder

When the box is tickedClick on a check box to tick it and thereby enable that function., the optimiser is authorised to group together (where possible) the machining operations that use the same tool. In this way it is possible to reduce the number of tool change operations and the time required to start up and shut down the electrospindle.

Arrange on the basis of tool direction

SortNV

When the box is ticked, the optimiser is forced to group together boring operations according to their working direction.

Optimise the first boring operation

DrillOverAll

When the box is ticked, the optimiser is forced to group together the consecutive boring operations, optimising them so as to make the least possible number of descents

Tools

This section is visualised by selecting the By tool option in the Type of optimisation data field.

List of tools arranged

OrderedTools

List of tools used in the machining operations.

This field is used to define the tool sequence to be used in order to arrange the machining operations; the machining operations are then performed on the basis of this sequence.

Arrange the tools using the relevant arrow buttons at the top.

Types of machining operations

This section is visualised by selecting the By type of machining operation option in the Type of optimisation data field.

List of machining operations arranged

OrderedWorkTypes

List of the machining operations contained in the document.

This field is used to define the machining operation sequence. Arrange the operations using the relevant arrow buttons at the top.

Tool change options
Sections together

SectionTog

This field cannot be modified. The box must be ticked.

Unload spindles

UnloadOutlets

When the box is ticked, the tool change device is forced to take the tool out of the spindle at the end of the programmed machining operation.

Group together tool changes

GroupTch

Enables (box ticked) or disables the grouping of the tool change operations.

Through machining operations at the end

SortPassThrough

Enables (box ticked) or disables the optimisation criteria of the through and non-through machining operations. When the box is checked, the optimiser first carries out the non-through machining operations and then the through machining operations.

Various data
Enable SW limits

EnableSWLim

Enables (box ticked) or disables the spindle check and makes sure they comply with the software limits. To make this test available, enable the field by clicking on the check box.

Best descent

BestHits

Used to make a selection from the several possible combinations of spindles that can be used in a single boring descent. In this case, the CAM, finding more than one spindle combination, can have the chance to select the best one.

Multi-descent tolerance

OptimTollMultiRun

Used to indicate the possible difference between two tools that can be used in a single boring and/or milling descent. In this case, the CAM, finding more than one spindle combination, can decide which tools to use in one moment according to the tolerance limit among the bit lengths.

Enable TP multiple downward run

OptimEnableMultiRunTP

Enables (box ticked) or disables simultaneous machining by two electrospindles.

Vertical bore lead

Enables (box ticked) or disables the following data field.

Value of vertical bore lead

FASTVERTBORINGSVALUE

Used to define the distance (mm) from the piece in order to make a smoother, faster movement between one machining operation and the next. In this way, the subsequent tool movement is brought forward.
