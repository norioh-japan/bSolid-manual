Rotation speed

REM_RPM

Tool rotation speed (rpm).

Machining speed

REM_Feed

Advance speed of the tool during the machining operation.

% Lead-in speed

REM_PercFeedIn

Percentage for increasing or reducing the tool advance speed as it enters the piece. This valuePredefined / pre-set. Standard information useful for the control, if the requested information is not entered. affects the speed defined in the Machining speed data field.

For example, if the tool has an advance speed of 3000mm/min and you want to reduce this to 1500mm/min during the entry phase, enter the number 50 in this field (3000 x 50% = 1500).

% Lead-out speed

REM_PercFeedOut

Percentage for increasing or reducing the tool advance speed as it emerges from the piece. This value affects the speed defined in the Machining speed data field.

Z safety position

REM_ZSic

Z position for establishing the distance that the tool must maintain from the piece during unit movement/release operations, in order to avoid any collisions with the table elements or the piece itself.

Z pushing position

REM_ZAcc

Z position for establishing the distance of the tool from the initial machining point. The tool arrives (at maximum speed) at the position established in this data field, then slows down to reach the initial machining point.

Finish type

REM_PAR_TYPE

Strategy type to generate the path to follow during the machining operation.

Select type by clicking one of the following options:

Concentric in Z

Concentric path with fixed distance in Z between one line and the next.

Parallel in Z

Path using parallel lines.

Working depth

REM_PAR_DEPTH

Depth of the machining operation.

  •    For surfaces created starting from the piece, the start Z position for depth calculation is the active face of the piece.
  •    For surfaces created without the reference piece, the start Z position for depth calculation is the one given by the coordinate system.
Z step

REM_PAR_ZSTEP

Depth of each movement. Enter the value of the depth of each single movement.

• See the example in figure 2.

This value is linked to the one defined in the "Number of movements in Z" data field; the modification of either of them leads to the automatic updating of the other one.

For example, if you set a machining operation with a depth of 30 millimetres (ref. A) and a step of 10 millimetres, the tool will make three movements, each 10 millimetres deep, until it reaches a depth of 30 millimetres. A value of 3 is automatically entered in the "Number of movements in Z" data field.

Finishing width

REM_PAR_DEPTHOFCUT

Size of the area to be finished off, starting from the tool path.

  •    If you enter a value of zero (ref. A, figure 3), the system will automatically calculate the path and the area to be machined; in this case, the cleaning operation may not be very precise and excess material may be left behind.
  •    If you enter a value higher than zero (ref. B, figure 3), the system is forced to consider a larger machining area; in this case, several movements will be made in order to fully remove all the excess material.

Machining mode

REM_PAR_WORKWAY

Method by which the tool moves along the paths specified in the "Finish type" data field.

This data field is visualised by selecting the "Parallel in Z" option in the "Finish type" field.

One Way

Each single movement is separate from the following one, and the tool ascends.

Zig Zag

The movements are all connected.

Machining direction

REM_PAR_WORKINOUT

Direction in which the tool performs the machining operation when the concentric or spiral paths are activated in the "Finish type" data field.

From in to out

The machining operation begins from the inside and moves outwards.

From out to in

The machining operation begins from the outside and moves inwards.

Enable number of passes in Z

SV_PAR_ENABLE_ZNSTEP

Enables (box ticked) or disables the "Number of movements in Z" data field.

Number of movements in Z

REM_PAR_ZNSTEP

Number of stages affecting the depth of the programmed machining operation.

This value is linked to the one defined in the "Z step" data field; the modification of either of them leads to the automatic updating of the other one.

For example, if you set a boring with a depth of 30 millimetres (ref. A, figure 2) and enter the number 3, the machining operation will be split into three parts with a depth of 10mm each, until the tool reaches the depth indicated in the "Working depth" field. A value of 10 is automatically entered in the "Z step" data field.

Start Z

REM_PAR_ZSTART

Machining operation start point (Z position).

The zero point of the Z axis is calculated in two different ways:

  •    For constructions which start from the reference piece, zero point is calculated starting from the face of the piece selected prior to applying the machining operation.
  •    For constructions which use the construction planePlane on which a flat geometry is constructed. The default plane is the XY plane., without a piece, zero point is set in the coordinates system.

The value shown in this field is subtracted from the one shown in the "Working depth"data field to set the total depth of the machining operation.

% Feed first pass

REM_PAR_FIRSTSTEP

Percentage to reduce the tool forward movement speed during the first movement or during all movements (according to the type of finish selected), that can be calculated according to the total tool forward movement speed.

Priority to passes in Z

REM_PARA_USEPLANE

Enables (box ticked) or disables the creation of the machining operations, taking into account the number of depth movements. In other words, if two geometries have been programmed and their implementation is divided into two movements, the tool will perform the first movement on both geometries, and then the second movement (if the box is ticked).

Optimisation of passes in Z

REM_PAR_ZSTEPOPT

Enables (box ticked) or disables the automatic calculation of the steps in Z, so that the tool descends evenly.

When the box is enabled, the "Eliminate release points between movements" field is active.

Uniform distance between passes

REM_PAR_CONSTANT

Enables (box ticked) or disables the automatic insertion of an even distance between the movements.

In this case, the value indicated in the "Distance between movements" data field is corrected slightly in order to ensure uniform paths.

The activation of this data field is connected to the type of option selected in the "Finish type" data field.

Eliminate release points between movements

REM_PARA_ZSTEPNOLINK

Disables (box ticked) or enables the withdrawal of the tool from the piece between one movement and the next.

When the box is ticked, the tool performs all operations one after another, without leaving the piece.

When the box is disabled, the "Z safety release points" field becomes active.

Z safety release points

REM_PARA_SVINCZSIC

Enables (box enabledClick on a check box to tick it and thereby enable that function.) or disables the positioning of the tool at the safety position.

When the box is ticked, the tool is forced to move to the safety position before each movement.

Profiling passes

REM_PAR_EDGEPASS

Enables (box ticked) or disables the machining operation related to the programmed profile.

Use surface profile

REM_PARA_USEPROFILE

Enables or disables the calculation of the machining operation path starting from the centre of the tool. If this box is enabled, the machining operation path stays within the perimeter of the surface to be machined (without exceeding it). If the box is disabled, the machining operation path extends outside of the perimeter of the surface.

For the description, see the "Use surface profile" data field of the finishing.

Remove on surface profile

REM_PARA_WRKPROFILE

Enables or disables the calculation of the machining operation path according to the radius of the tool. If this box is enabled, the tool extends outside the profile by a value equal to its radius.

For the description, see the "Remove on surface profile" data field of the finishing.

Supplementary profile offset

REM_PARA_PROFOFFSET

Distance from the perimeter of the machinable surface, so that the path of the machining operation leaves a margin (negative value) or extends outside of the profile of the surface (positive value).

For the description, see the "Supplementary profile offset" data field of the finishing.

Solid subdivision tolerance

REM_PARA_TOLL

Tolerance that defines the precision of the curvature of the created surfaces.

The value to be entered refers to the chord height necessary to ensure that the arc will be more or less precise. A value less than 0.05 results in more precise arcs; the calculation will be slower; a value greater than 0.05 results in less precise arcs; the calculation will be quicker.

208. Chord height (h)
Solid machining accuracy

REM_PARA_PREC

Index for defining the precision of the machining operation on curved surfaces, in order to calculate the tool trajectories. Values less than 0.05 result in a more precise machining operation.
