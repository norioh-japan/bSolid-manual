2. ルール設定をエクスポート他のソフトウェアプログラムが読み込めるファイル（ドキュメント）の生成を伴う操作。しますカードをクリックします。

ルールカードは、ノード定義については、ツリーリストを参照します。機械のセクション結果のエクスポートのデータフィールドデータまたはコマンドを含む所定のエリア（データフィールドまたはコマンドフィールド）。ルールをエクスポートしますのアイコンをクリックして開くこともできます（参照カード一般設定）。

注：

このカードのデータは、Biesseの技術者のみが使用できます。

これにより、プロジェクト処理結果に関する情報をエクスポートするルールを定義して、処理済データをマシンに送信する場合に特別なテキストファイルに保存することができます。

作成可能な4タイプのファイルがあります。

• カードピースで定義されたデータを報告するファイル。
• カード残余物（ネスティング）加工作業中に排除される、再利用可能な残ったピース。で定義されたデータを報告するファイル。
• カードスキーマで定義されたデータを報告するファイル。
• カード統計で定義されたデータを報告するファイル。

ファイルオプション
ファイルに追加します

エクスポート中にファイル内のデータの上書きを実行することを、有効（ボックスにチェック）または無効にします。

ボックスをチェックすると、ファイルは上書きされたままですが、前回のデータもクリアされません。この場合、新規の情報はその内部の、前回の情報の下に書き込まれます。

ファイル名

保存するファイル名。

このフィールドでは、左側エリアに存在するデータを追加することも可能です。

拡張子

保存するファイルの、ドットで始まらない拡張子（例えば、CSV）。

行き先フォルダ

送信フェーズ中にエクスポートされたファイルが保存されるフォルダを定義するために使用します。

画像フォルダ

送信フェーズ中にエクスポートされた画像のファイルが配置されるフォルダを定義するために使用します。

保存された画像は拡張子.PNGを有し、上書きできません。システムにより別のデータを含む新規の画像が常に作成されます。

フォーマットの選択
日付と時刻のフォーマット化を有効化します

保存されたファイルに定義された情報の、日時フォーマットを定義可能にすることを、有効（ボックスにチェック）または無効にします。

この設定は、ファイルを保存する毎に、エクスポートされたファイルの名前に日時を表示させるのに非常に便利です。この方法で、異なる日付の多くのコピーが作成され、ファイルは決して上書きされません。

日付

ドロップダウンメニュードロップダウンボタンをクリックすると、垂直または水平方向に広がるオプションまたは手段のリスト。から日付のタイプを選択します。M（月）、D（日）、Y（年）。

注：

デフォルトでは、日付はスラッシュ（/）で分割されています。データフィールド"ファイル名"に日付を入力すると、許可されていない文字は自動的に置き換えられますが、それは、オペレーティングシステムではファイル名に以下の文字を入れることができないからです\ / : * ? " < > |。

時間

ドロップダウンメニューサブメニュー（特定の機能/作業を実施できるようにするオプションのリスト）を生成するコマンド（マウスで選択する）のリスト。から時間のタイプを選択します。H（時）、m（分）、s（秒）。

注：

デフォルトでは、時間はコロンで分割されています。データフィールド"ファイル名"に時間を入力すると、許可されていない文字は自動的に置き換えられますが、それは、オペレーティングシステムではファイル名に以下の文字を入れることができないからです\ / : * ? " < > |。

フォーマット

最終フォーマットを表示します。このフィールドには、カスタムの日時フォーマットを入力できます。

例：

MM-yyy_HH-mm-ssを入力してフィールド"ファイル名"にデータ"日付と時間"を設定すると、保存されたファイル名には、頭文字ART_ の次に月、年、時が入ります。

数のフォーマット化を有効化します
