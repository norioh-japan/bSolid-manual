Processing the project

The project can be processed either from the project table or from within the single project.

1. Click on the button.
2. Select the table row showing the name of the project, then click on the button.

If you are working within a project, you can use the same button to process it without returning to the project table.

If the project has been processed correctly, the orange LED will appear in the cell of the Status column of the project row.

Otherwise, you will see a red cross with the Processing failed text.

If modifications are made after processing a project row, the orange LED will turn grey ; in this case, you must process the project again.

Description of the status LEDs:
Can be processed

Grey LED: project still to be processed.

Being processed

Yellow LED: project being processed.

Processed

Orange LED: project processed but not yet transmitted.

If modifications are made while in this status, the LED will turn grey to show that the project needs to be processed again.

Being transmitted

Light blue LED: project being transmitted.

Transmitted

Blue LED: project transmitted to the machine.

 
Transmitting the processed project
1. Click on the button.
2. Processing the project .
3. Select the table row showing the name of the processed project, then click on the button.

If the project has been transmitted correctly, the blue LED and the text Transmitted will appear in the cell of the Status column of the project row.

Generating the programs automatically with the rejected machining operations

During the processing of the nesting project, if the system finds machining operations not compatible with the machine being used, it eliminates them from the final result and does not generate them in the programs produced.

To ensure that these machining operations are not lost, you can create “complementary programsDocument containing the machining operations found in the nesting projects, but that the machine cannot carry out because it is not configured. During project processing, if the system finds machining operations that are not compatible with the machine, it creates parallel programs and inserts these operations in them.”.
To do this, proceed as follows:

1. Open the project.
2. Click on the Processing parameters card.
3. Click on the Project parameters data field.
4. TickClick on a check box to tick it and thereby enable that function. the Generation of supplementary programs box in the specific project parameters window.
5. Confirm the operation carried out.
6. Save the nesting project.
