Work list evolution mode

Used to define the type of execution of the work list.

Choose one of the following options from the drop-down menuList of options or instruments which expand vertically or horizontally when you click on the drop-down buttons.:

Standard

The programs are executed based on the sequential order of the work list rows.

For example: suppose we have 3 rows:

Program

Quantity

Rout01

5

Rout02

3

Rout03

2

In this case the Rout01 program is run 5 times, then the Rout02 program 3 times and finally the Rout03 program 2 times.

Consumption

One program is performed at a time for each row in rotation until the depletion of the individual quantities.

For example: suppose we have 3 rows:

Program

Quantity

Rout01

3

Rout02

2

Rout03

2

In this case, the system performs the Rout01 program once, then the Rout02 program once and finally the Rout03 program once. Therefore it starts again from the first row rerunning Rout01, then Rout02 and finally Rout03.

Since the Rout01 program has to be run 3 times, it starts again, only that the subsequent programs are no longer executed because the quantities have ended.
