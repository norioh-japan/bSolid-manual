Information about the environment

The List environment allows you to manage the work lists.

The work list is a card containing a list of machining programs (dSolid, isoAcronym of the "International Standardisation Organisation", that determines technological and industrial standards. File format containing the programming language of the numerical control machine., bpp, etc.) arranged in sequential order, to be executed on the machine.

Modifications can be made to the parameters and/or variables of each single program in the list; these modifications are applied within the list only, and do not affect the structure of the source file.

Overview of the environment

At the launch, the List environment shows the DatabaseData bank. Stored data arranged in an organised manner so that they are readily available using a search and management program. card with the table of all the work lists recorded in the database.

The visualisation of this environment is multi-card. This means that several cards can be opened simultaneously, so you can manage and control several work lists.

The colours of the environment

The colours of the work list rows

  •    Row with a green border

Indicates that the program has been transmitted and will be machined.

  •    Row with a dark green background

Indicates that the program is being executed and the machine is working.

  •    Row with a light green background

Indicates that the program has been completed.

  •    Row with a red background

Indicates an error in the row. For example: Quantity, Piece count, incorrect variables or optimisations.

The colours of the data field Piece count

  •    Green

Time calculated.

  •    Red

Time not valid (program data may have been modified). In this case, launch the time calculation procedure again.
