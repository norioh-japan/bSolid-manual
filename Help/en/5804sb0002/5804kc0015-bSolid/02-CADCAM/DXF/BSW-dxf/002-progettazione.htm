Designing the layer profiles and syntax

Description of the topics

The following topics explain how to design the Biesse Works DXFAcronym of "Drawing Interchange Format". Type of two-dimensional Vector graphics file used as a graphic data exchange format between the various CAD systems. files to be imported.

The design of the DXF files, and the relative layerLevels of a document created to carry out a drawing. syntax, must respect the default keys indicated in the relevant table of the "Layer parameters" section of the DXF definition dialogue box.

Creating the piece

The piece must be geometrically represented by a closed rectangle, using the commandTool designed to perform a specific function/operation. For example, the “Print” command allows you to print a document. for generating closed polygons, lines or polylinesSet of consecutive straight lines, connected to each other at the ends..

To create the piece, use one of the following methods:

  •    using a single rectangle, to which the three dimensions (width, thickness, height) are assigned. The dimensions relating to the length of the piece along the X and Y axes are provided directly by the geometry, whereas the thickness must be inserted in the layer associated with the rectangle.
If the depth is not specified it will be taken to be zero.
Example:

To draw a piece with dimensions 1000 x 300 x 30.20 (mm), it will be necessary to associate a rectangle 1000mm wide and 300mm high with the layer TCHW0B8D3020:

TCH is the prefix identifying the BIESSE layer.

W0: W = parameter key "Face"; 0 = face number.

B8: B = parameter key "Work"; 8 = identifies the piece.

D3020: D = parameter key "Depth"; 3020 identifies the valuePredefined / pre-set. Standard information useful for the control, if the requested information is not entered., i.e. 30.20 mm (the last two numbers represent the obligatory decimals).

346.
  •    Via several rectangles, to create the standard sides of the piece, or lines inside the zero side associated with rectangles in order to create the general sides of the piece.

A layer must be associated with each side created.

See also:
• Creating and naming standard panel faces
• Creating and naming generic faces.
NOTES:

Standard sides are the 6 sides of the piece: the top, bottom and side faces. Generic sides are the sides created by the user.

For piece recognition, always insert the parameter key (e.g. B) and the value 8 (e.g. B8) in the layer associated to the rectangle describing the single face "Work".

Creating and naming generic faces

A generic face must be represented by a line or an arc, according to whether it is a straight or circular face. The arcs and lines must be drawn inside face 0, in the exact point in which a new side of the piece is to be created.

The layer must indicate the key for the generic face and the identification number for the new side, and it must provide the height of the side. If the height is not indicated, the height of the panel will be used.

Example:

To create a general linear face associated with a line drawn inside face 0 of the piece with a height of 30mm, the face must be associated with the layer TCHWG6B8HF3000:

TCH is the prefix identifying the BIESSE layer.

WG6; WG = key of the "Generic face" parameter; 6 = number of the face (ref. 6 figure 5).

B8; B = parameter key "Work"; 8 = identifies the piece (obligatory).

HF3000; HF = parameter key "Face height"; 3000 identifies 30mm (the last two numbers represent the obligatory decimals).

The surface of the machinable side of the piece is determined by the direction in which segment 6 (shown in figure 5) is drawn.

350.

The direction of the segment thus has a start point P1 and an end point P2.
The part of the segment that identifies the machinable face (ref. A and B, figure 6) is the one that goes from the start point P1 to the end point P2 (on the left of the segment, in direction P1-P2). It's possible to force one vertex of the segment by drawing a circle around it, defining it as the start point. This operation cancels the order in which the corners of the segment were drawn, so that the machinable face will always be recognisable.

352. Forced setting of the segment vertex to establish the start point P1, i.e. the machinable surface of face 6 (ref. C).

When an arc is created to represent a generic curved face, the system interprets it as if it were created anticlockwise irrespective of the direction of what has been created.
For this reason it is advisable always to use forcing, i.e. the circle to establish the initial vertex of the arc (figure 8).

Let's suppose we want to create an arc going from P1 to P2, figure 8, in a clockwise direction. The system will interpret P2 as the first vertex of the arc and P1 as the second vertex. To establish the vertex P1 as the first part of the arc, it is therefore necessary to using forcing on P1 to mark it with a circle.

Creating machining operations on the general faces

If the generic face is to contain machining operations, it must not only be drawn as a line/arc inside the piece, but also as a horizontal rectangle located under face 0 of the piece.
In this case different layers must be assigned to the line/arc identifying the face and to the rectangle.

To create generic faces with identification numbers higher than 5, always use parameter key "Generic face", (e.g. WG) and draw the rectangle that represents them under face 0 or adjacent to it, with corner 2 facing towards the bottom left hand corner of the rectangle.

The dimensions of the rectangle drawn to represent the generic face do not have to coincide with those of the segment (line or arc) associated with them, as this value will be detected from the segment geometry and layer.

Example:

If the segment (line or arc) that describes the position of the face has the layer called TCHWG6B8HF3000, then also the layer of the rectangle that represents it (and on which the machining operations will be carried out) must have the same syntax, with or without the data concerning the face height (HF3000).
This means that two types of code can be used: TCHWG6B8HF3000 or TCHWG6B8:

TCH is the prefix identifying the BIESSE layer.

WG6; WG = parameter key "Generic face"; 6 = face number.

B8; B = parameter key "Work"; 8 = identifies the piece (obligatory).

HF3000; HF= parameter key "Face height"; 3000 identifies the value, i.e. 30mm (the last two numbers represent the obligatory decimals).

Creating standard borings

Bores can be represented in two different ways:

  •    by a circle, which can be drawn directly on the standard or generic faces created as rectangles.
Vertical boring operations represented by circles

With this method, bores can be made with the vertical spindle on face 0 or face 5 of the piece. In this case, the bore must be drawn inside the rectangle representing the required face.

The diameter of the bore is given by the diameter of the circle, whereas the depth must be specified in the layer. If the depth is not specified it will be taken to be zero.

The bore applicationA synonym of "program" - software with the specific task of managing data, graphics, etc. point coincides with the centre of the circle.

Example:

To create a bore with a depth of 25.5mm on face 0 of the piece, associate the bore drawing with the layer TCHW0B2D2550:

TCH is the prefix identifying the BIESSE layer.

W0; W = parameter key "Face"; 0 = standard face number.

B2; B = parameter key "Work"; 2 = identifies the boring.

D2550; D = parameter key "Depth"; 2550 = identifies the value, i.e. 25.50mm.

Horizontal boring operations represented by circles

This method can be used to create bores with the horizontal spindle on sides 1, 2, 3 and 4 of the piece, or on the general sides. In this case, the bore must be represented with a circle drawn on one of the side faces of the piece.

354.

The diameter of the bore coincides with the diameter of the circle, while the depth must be specified in the layer.

Example:

To create a bore with a depth of 25.5mm on face 3 of the piece, associate the bore drawing with the layer TCHW3B2D2550:

TCH is the prefix identifying the BIESSE layer.

W3; W = parameter key "Face"; 3 = standard face number.

B2; B = parameter key "Work"; 2 = identifies the boring.

D2550; D = parameter key "Depth"; 2550 = identifies the value, i.e. 25.50mm.

  •    by creating a rectangle or a closed polygonal line. In this case the rectangle must be created directly on face 0 of the piece, and indicates a bore to be applied to the relative side face, for example face 4 of the piece.
Horizontal boring operations represented by rectangles

This method can be used to create bores with the horizontal spindle on sides 1, 2, 3 and 4 of the piece.

When a bore is created using a rectangle, its diameter and depth are obtained from the geometry.

  •    The diameter is indicated by the length of segment A (figure 10) performed on segment C (figure 10), which represents face 1 on which the bore is made.
  •    The depth is indicated by the length of segment B (figure 10).
  •    The application point is the average point of the segment, representing the bore diameter.

Creating angular borings

To create tilting bores you must define the application point and the AR and AZ angles.
The AR angle represents the spindle axis rotation on the xy plane.
The AZ angle represents the tilt angle of the spindle rotation axis in relation to the xy plane.

Angular bores can be illustrated by a circle, a rectangle or a closed polygonal line on the 0 face.

• "Angular boring represented by a circle"
• "Angular boring represented by a rectangle "
Angular boring represented by a circle

If the angular bore is represented by a circle, the depth and the angles AR and AZ must be specified in the layer.

The pre-set value for angles AR and AZ is zero.

Example:

To create a bore on face 0 with a depth of 10mm, AR angle 45° and AZ angle 30°, associate the bore drawing with the layer TCHW0B2D1000AR45AZ30.

TCH is the prefix identifying the BIESSE layer.

W0: W = parameter key "Face"; 0 = face number.

B2; B = parameter key "Work"; 2 = identifies the boring.

D1000; D = parameter key "Depth"; 1000 identifies the value, i.e. 10mm (the last two numbers represent the obligatory decimals).

AR45; AR = parameter key "AR"; 45 = angle value.

AZ30; AZ = parameter key "AZ"; 30 = angle value.

Angular boring represented by a rectangle

If the angular bore is represented by a rectangle, the diameter is given by the length of the first side, the one between corners 1 and 2 (ref. A, following figures), and the depth is given by the length of the second side, the one between corners 2 and 3 (ref. B, following figures).

The tilt angle is between the second side and the positive X-axis. In this case the corners of the rectangle must be drawn in the correct order. If the AZ angle is other than zero, this must be specified in the layer.

To specify the side of the rectangle that represents the diameter, it is possible to use a line drawn on this side (figure 14).

Creating the cuts

A cut is represented geometrically by a line. The layer associated with the line is used to specify not only the type of machining operation and the face on which it is carried out, but also the cut depth and other technological parametersData allowing you to specify how the created drawings should be reproduced on a piece..

If the depth is not specified it will be assumed to be zero.

Cuts on side faces are also managed.

Example:

To create a cut on face 2 (15mm deep and 2mm wide), associate the layer TCHW2B3D1500TH0200 with the drawing:

TCH is the prefix identifying the BIESSE layer.

W2; W = parameter key "Face"; 2 = face number.

B3; B = parameter key "Work"; 3 = identifies the cut.

D1500; D = parameter key "Depth"; 1500 identifies the value, i.e. 15mm (the last two numbers represent the obligatory decimals).

TH0200; TH = parameter key "Thickness"; 200 = identifies the value, i.e. 2mm.

Creating the milling

A milling operation can be represented geometrically by lines, arcs, circles, and closed or open polygonal lines.

The layer associated with the geometric elements is used to specify not only the type of machining operation, but also the milling depth and other technological parameters (tool descent speed, tool type, etc.).
If the depth is not specified it will be assumed to be zero.

Milling operations on the side faces are also managed.

Example:

To create a milling operation (10mm deep) on face 1, associate the layer TCHW1B1D1000 with the drawing:

TCH is the prefix identifying the BIESSE layer.

W1; W = parameter key "Face"; 1 = face number.

B2; B = parameter key "Work"; 1 = identifies the milling.

D1000; D = parameter key "Depth"; 1000 identifies the value, i.e. 10mm (the last two numbers represent the obligatory decimals).

Ways of creating the milling operations
  •    To generate a milling operation containing several geometric elements, associate a single layer - for example TCHW0B1DP1000 ref. A - with the whole drawing.
  •    To ensure that each single geometric elementAn abstract element associated with specific properties: the point, line, plane and space are all geometric elements. is recognised as a single milling operation, you must create three layers (one for each geometric element):
• TCHW0B1DP1000 associated with drawing B;
• TCHW0B1PT1 associated with drawing C;
• TCHW0B1DP2000 associated with drawing D.
