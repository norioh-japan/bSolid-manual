Rules for generating the DXF files to be imported

The string (syntax) of the layer of the DXF files created with general CADAcronym of "Computer Aided Design". Type of editor for three-dimensional (3D) and/or two-dimensional (2D) graphic design. programs must consist of keys and numbers:

  •    Keys

Used to define the type of geometry and certain technological data items. Can be personalised in the DXF definition dialogue box.

  •    Numbers

Used to define the geometric and technological values.

The measurement units of the DXf files to be imported

It is possible to importInsert, in an open document (file), information created with another program and in a different format from the one currently being used. DXF files expressed using the following units of measurement:

  •    millimetres;
  •    inches.

If the geometric data are expressed in millimetres, the unit of measurement for the data in the layer must be millimetres. If the geometric data are expressed in inches, the unit of measurement for the data in the layer must be inches.

The two units of measurement must not be used simultaneously.

Using the blocks or groups

Blocks can only be used to represent bores.

In order for a block (i.e. grouped geometries) to be recognised as a bore, you must fill in the "Block parameters" table of the DXF definition dialogue box, and define the data concerning the diameter, depth and tool type.

A block indicating a bore on a side face must have a layer specifying that face.

Management is totally similar to that of horizontal bores represented using a rectangle.
