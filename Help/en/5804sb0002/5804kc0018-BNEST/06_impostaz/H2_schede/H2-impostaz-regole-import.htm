654. Button list

A. Opens a rules file (XML) and populates both rules tables (figure 1).
B. Saves the rules file created, or overwrites the data of the one currently open.
C. Creates a copy of the rules file currently open, giving it a different name.
D. Imports the information saved in the BNIMP format file and populates the two rule tables.
E. Exports the information defined in the rule tables to a BNIMP format file.
F. Shows the dialogue box for defining which rule to delete.
G. Opens the CSV file and displays the data in the relevant area (ref. C, figure 1).
H. Automatically populates the left table (ref D, figure 1) using the data present in the CSV format file opened in the appropriate area (CSV viewing area ).
I. Connects the open CSV file with the data defined in the left table.
J. Disconnects the open CSV file of the data defined in the left table.
K. Checks whether the rules file contains any errors.
L. Inserts a new row in the table.
M. Eliminates the selected row from table .
N. Empties the table, deleting all the compiled rows.
O. Moves the selected row, sliding it over the previous one.
P. Moves the selected row, sliding it underneath the next one.

Date format

Used to define the date format used in the file to be imported.

CSV with discriminator

Enables (box ticked) or disables the presence of the column Discriminator in the file to be imported.

If the file to be imported has the column Discriminator, enable the box.

The enabled box allows to import text files that contain data from multiple archives and not a single archive; for example, the data to populate both the articles archive as well as the materials archive.
