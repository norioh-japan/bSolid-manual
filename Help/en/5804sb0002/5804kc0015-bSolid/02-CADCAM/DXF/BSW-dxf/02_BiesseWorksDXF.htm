Rules for generating the DXF files to be imported

The string (syntax) of the layer of the DXF files created with general CADAcronym of "Computer Aided Design". Type of editor for three-dimensional (3D) and/or two-dimensional (2D) graphic design. programs must consist of keys and numbers:

  •    Keys

Used to define the type of geometry and certain technological data items. Can be personalised in the DXF definition dialogue box.

  •    Numbers

Used to define the geometric and technological values.

Observe the following rules:

  •    The syntax of the layer must begin with the key TCH.
  •    The syntax of the layer may contain linear dimensions relating to the depth, thickness, diameter and height of the piece side, and they can be expressed in millimetres or in inches.

Decimal figures must always be indicated. When the values are expressed in millimetres it is necessary to specify 2 decimal figures: when they are expressed in inches it is necessary to specify 4 decimal figures.

For example:

• for a depth of 30.5mm, write D3050, where 50 is the 2-figure decimal valuePredefined / pre-set. Standard information useful for the control, if the requested information is not entered.;
• for a diameter of 5mm, write D500, where 00 is the 2-figure decimal value;
• for a diameter of 2.5 inches, write 25000, where 5000 is the 4-figure decimal value.
  •    The syntax of the layer may contain linear velocities relating to the working speed and descent speed, and they can be expressed in mm/min or in inches/min.

When indicating the value for these speeds, if they are expressed in mm/min the decimals need not be indicated, whereas if they are expressed in inches/min it is necessary to indicate 2 decimal figures.

For example:

• to indicate a tool feed speed of 12000mm/min, write F12000;
• to indicate a tool feed speed of 5000 inches/min, write F500000, where 00 is the 2-figure decimal value.
  •    The syntax of the layer may contain the rotation speeds, which are expressed as rpm.
  •    The alphanumericMade up of numbers and letters. values must start and end with the separator character.

For example: the characters MN$MACRO1$ within a layer indicate that the parameter MN has a value of MACRO1.

The measurement units of the DXF files to be imported

It is possible to importInsert, in an open document (file), information created with another program and in a different format from the one currently being used. DXF files expressed using the following units of measurement:

  •    millimetres
  •    inches

If the geometric data are expressed in millimetres, the unit of measurement for the data in the layer must be millimetres. If the geometric data are expressed in inches, the unit of measurement for the data in the layer must be inches.

The two units of measurement must not be used simultaneously.

Using the blocks or groups

Blocks can only be used to represent bores.

In order for a block (i.e. grouped geometries) to be recognised as a bore, you must fill in the "Block parameters" table of the DXF definition dialogue box, and define the data concerning the diameter, depth, tool characteristics, etc.

The block must be drawn directly on the destination face; the face must be specified in the layer.

It is not possible to draw a block on face 0 of the panel and then specify in the layer that the bore is on another face.
